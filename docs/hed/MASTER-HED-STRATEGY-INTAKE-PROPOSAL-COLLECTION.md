## The 4 Structural Failure Modes This SOP Eliminates

1. **Invisible Queue** — requests arrive via email, Slack, hallway conversations
2. **SLA Ambiguity** — no documented expectations
3. **No KB Leverage** — re-answering the same questions
4. **No Feedback Signal** — no systematic learning mechanism

## Seven-Stage Lifecycle

```
STAGE 1: REQUEST RECEIVED
STAGE 2: INTAKE FORM COMPLETED (within 2 hours)
STAGE 3: TRIAGE (classify → check KB → estimate effort → RPS score → confirm SLA)
STAGE 4: ASSIGNMENT & EXECUTION
STAGE 5: INTERNAL REVIEW
STAGE 6: DELIVERY (with MCR-ID + confidence level + caveats)
STAGE 7: FEEDBACK + ARCHIVE (T+5 day follow-up with 3 questions)
```

## Request ID System

```
MCR-{YYYY}-{NNNN}

Examples:
  MCR-2026-0001
  MCR-2026-0247
```

MCR = Mike CI Request

## RPS Algorithm (Request Priority Score)

```
RPS = (business_impact × 0.35) + (deadline_urgency × 0.30) +
      (requestor_seniority × 0.20) + (strategic_alignment × 0.15)

Range: 0 – 100
```

## SLA Table

| Request Type | Standard SLA | P0 SLA | Escalation Trigger |
|---|---|---|---|
| Quick Fact | 24 hours | 4 hours | 20 hours no delivery |
| Competitor Update | 48 hours | 4 hours | 40 hours |
| Battle Brief | 48 hours | 4 hours | 40 hours |
| Deep Analysis | 5 business days | N/A | Day 4 no delivery |

**Hiring trigger:** P(utilization > 80%) > 0.55 or mean queue > 20 hours
